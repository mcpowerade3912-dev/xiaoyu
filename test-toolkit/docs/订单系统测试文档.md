## 第一章 测试概述

### 1.1 测试范围

本测试文档覆盖订单管理系统的全部 6 大模块、20 个接口：

| 模块 | 接口数 | 认证要求 |
|------|--------|---------|
| 用户认证 | 3 | login/register 免登录，logout 需登录 |
| 商品查询 | 1 | 免登录 (/goods/**) |
| 管理后台 | 4 | 需管理员Token (role=1) |
| 购物车 | 3 | 需登录 |
| 订单管理 | 5 | 需登录 |
| 地址管理 | 6 | 需登录 |

### 1.2 认证与权限体系

```
请求流程:
1. 客户端发送请求，Header 携带 Authorization: Bearer <token>
2. JwtAuthInterceptor 拦截所有路径 (/**)
3. 白名单放行: /user/register, /user/login, /error, /goods/**
4. 其他路径: 校验Token有效性 → 查询用户状态 → 判断角色权限
5. /admin/** 路径: 仅 role=1 (管理员) 可访问，否则返回 403
```

### 1.3 统一返回格式

```json
{
  "code": "200",
  "msg": "操作成功",
  "data": {}
}
```

**状态码规范**:

| code | 含义 |
|------|------|
| 200 | 操作成功 |
| 40001 | 参数校验失败 |
| 40002 | 用户名已存在 |
| 40003 | 用户不存在 |
| 40004 | 密码错误 |
| 40005 | 用户被禁用 |
| 500 | 服务异常 |
| 50001 | 服务器内部错误 |
| 401 | 未登录/Token失效 |
| 403 | 权限不足 |
